\subsection{Detailed Description}
Implements an image, a 2-\/dimensional array of color values.

An image can be used to store a color texture.

These color values are of type \hyperlink{classnoise_1_1utils_1_1_color}{Color}.

The size (width and height) of the image can be specified during object construction or at any other time.

The \hyperlink{classnoise_1_1utils_1_1_image_ab923c8098c27b51ac1aa2a79a324f6f8}{Get\+Value()} and \hyperlink{classnoise_1_1utils_1_1_image_ac99e2764bf63c6fa1fc2c4bec459d049}{Set\+Value()} methods can be used to access individual color values stored in the image.

This class manages its own memory. If you copy an image object into another image object, the original contents of the image object will be freed.

If you specify a new size for the image and the new size is smaller than the current size, the allocated memory will not be reallocated. Call \hyperlink{classnoise_1_1utils_1_1_image_a7cfa047d3eda1cb39ac75830fe6fd051}{Reclaim\+Mem()} to reclaim the wasted memory.

{\bfseries Border Values}

All of the color values outside of the image are assumed to have a common color value known as the {\itshape border value}.

To set the border value, call the \hyperlink{classnoise_1_1utils_1_1_image_a5ce5f632933cf6b38a3971c1ead5e186}{Set\+Border\+Value()} method.

The \hyperlink{classnoise_1_1utils_1_1_image_ab923c8098c27b51ac1aa2a79a324f6f8}{Get\+Value()} method returns the border value if the specified position lies outside of the image.

{\bfseries Internal \hyperlink{classnoise_1_1utils_1_1_image}{Image} Structure}

Internally, the color values are organized into horizontal rows called {\itshape slabs}. Slabs are ordered from bottom to top.

Each slab contains a contiguous row of color values in memory. The color values in a slab are organized left to right. These values are of type \hyperlink{classnoise_1_1utils_1_1_color}{Color}.

The offset between the starting points of any two adjacent slabs is called the {\itshape stride amount}. The stride amount is measured by the number of \hyperlink{classnoise_1_1utils_1_1_color}{Color} objects between these two starting points, not by the number of bytes. For efficiency reasons, the stride is often a multiple of the machine word size.

The \hyperlink{classnoise_1_1utils_1_1_image_a4349e07bc8d153d64c0a21a0e1062701}{Get\+Slab\+Ptr()} methods allow you to retrieve pointers to the slabs themselves.
